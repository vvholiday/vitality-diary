# 元气日记 — 产品需求文档（PRD）

**版本：** v0.1（MVP）
**日期：** 2026-07-22

---

## 一、产品背景与定位

### 1.1 产品概述

「元气日记」是一个帮助人们**记录和追踪日常抗衰老行为**的在线打卡工具。用户可以配置自己要做的保养项目（吃什么补剂、做什么运动、睡眠习惯等），每天打卡记录执行情况，并通过数据反馈看到自己的坚持和进步。

### 1.2 解决什么问题

- 抗衰老是一件「做了不一定马上看到效果，但不做以后一定后悔」的事
- 很多人知道该做什么，但缺少一个**简单、持续的工具**来记录和追踪
- 需要一个地方把零散的保养行为（吃补剂、护肤、运动、睡眠）**统一管理**起来

### 1.3 目标用户画像

- **年龄段：** 25-50 岁，开始关注抗衰老和健康管理的人群
- **行为特征：** 有一定的健康意识，尝试过各种保养方法，但缺乏系统记录的习惯
- **技术水平：** 会使用浏览器上网，能用邮箱注册账号

## 三、功能需求

| ID | 功能 | 说明 |
|---|---|---|

### 3.2 打卡核心功能（C端）

| ID | 功能 | 说明 |
|---|---|---|
| F-05 | 查看当日打卡 | 打开网站默认进入当天打卡页，按分类分组展示所有打卡项目 |
| F-06 | 打卡操作 | 每个项目可勾选"完成"，数值型项目还可填写具体数量（如30分钟） |
| F-07 | 保存打卡 | 一键保存当天所有打卡记录 |
| F-08 | 日期切换 | 顶部可切换到任意日期，查看和补打历史记录 |
| F-09 | 补打卡 | 只能补打**7天内**的记录（比如今天7月22日，最多补到7月16日），补打的记录会被标记为"补打卡" |
| F-10 | 每日评分 | 系统根据当天完成的项目的**分值**自动计算总分。比如有氧运动打卡得10分，抗阻运动打卡得15分，没有打卡的项目不得分，每日总分自动累加 |

| F-13 | 连续打卡天数 | 显示当前连续打卡天数（补打卡不计入） |

### 3.4 个人设置（C端）

**登录状态处理：**
- 用户登录前可以浏览主页面（看到项目列表、查看布局）

## 四、打卡项目的设计规则

每个项目的配置方式如下：

| 配置项 | 说明 | 示例 |
|---|---|---|
| 所属分类 | 这个项目属于哪个分类 | 运动 |
| 项目名称 | 打卡项目的名称 | 有氧运动 |
| 项目类型 | 开关型（做了/没做）或数值型（做了+填量） | 数值型 |
| 单位 | 数值型项目需要填单位 | 分钟 |
| 单次分值 | 完成这个项目获得的分数 | 10分 |
| 说明 | 可选的温馨提示 | 建议每周3-5次，每次30分钟 |
| 排序 | 在页面上显示的顺序 | 1 |
| 启用状态 | 是否在前端显示 | 启用 |

**关于分值的设计逻辑：**

每个项目都有一个"单次分值"。比如：

| 项目 | 单次分值 |
|---|---|
| 有氧运动 | 10分 |
| 抗阻运动 | 15分 |
| 拉伸 | 5分 |
| 喝够2升水 | 8分 |
| 11点前入睡 | 20分 |

如果用户今天完成了有氧运动 + 拉伸 + 早睡，总分 = 10 + 5 + 20 = **35分**。这样用户每天能看到一个数字化的"元气值"，比单纯的"做了/没做"更有激励感。你可以根据项目的重要程度来分配分值。

## 六、数据模型（简化版）

> 以下用最通俗的方式说清楚数据怎么存，不用管技术细节。

**1. 用户表（users）**

存谁在用这个网站，包括他们的角色（管理员/普通用户）。

**2. 分类表（categories）**

存打卡分类，比如"运动""饮食""睡眠"。

**3. 项目表（items）**

存具体的打卡项目，比如"有氧运动"。每个项目属于一个分类，有自己的**分值和单位**。

**4. 打卡记录表（check_ins）**

最核心的数据表。每一条记录 = 谁 × 哪天 × 做了哪个项目 × 做了多少。补打卡会有特殊标记。

**5. 每日评分表（daily_scores）**

每天每个用户的累计总分。每次打卡后自动更新，方便统计页面快速展示。这条数据是系统自动算的，用户不用管。

## 七、MVP 功能边界

### ✅ 包含

- 邮箱注册登录
- 管理员在后台配置分类和项目（含分值和单位）
- 用户日常打卡（开关 + 填数值）
- 日期切换和 7 天内补打卡
- 每日自动评分
- 日历视图看历史
- 个人统计（雷达图 + 趋势图）
- 连续打卡天数
- 管理员数据导出

### ❌ 不包含（以后再说）

- 社区/社交功能（排行榜、好友PK）
- 第三方登录（微信/Google）
- 微信小程序版本
- 消息推送/提醒
- 多人协作/团队
